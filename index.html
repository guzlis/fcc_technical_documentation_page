Intro to CSS

CSS (Cascading Style Sheets) is a fundamental technology in web development that controls the visual presentation of HTML elements on a webpage, allowing designers and developers to define styles such as layout, colors, fonts, and sizes, ultimately enhancing the overall look and feel of websites.

Required Know How
  • A basic understanding of the HyperText Markup Language (HTML)
  • A general understanding of the Internet and the World Wide Web
History of CSS

CSS (Cascading Style Sheets) has a rich history that spans several decades. It emerged as a solution to the growing complexity of styling web pages and played a crucial role in separating the presentation layer from the content layer, making web design more flexible and maintainable.

In the late 1990s, CSS1 was introduced as a specification by the World Wide Web Consortium (W3C). It provided a basic set of styling capabilities, allowing developers to define fonts, colors, and layout properties. However, browser support for CSS1 was inconsistent, leading to compatibility issues and limiting its widespread adoption.

To address these challenges, CSS2 was developed and released in 1998. It introduced new features like positioning, floats, and z-index, expanding the possibilities for web layout and design. CSS2 also introduced media types, enabling different stylesheets to be applied based on the device or medium used to view the content, such as print or screen.

The release of CSS3 marked a significant milestone in the evolution of CSS. Rather than a single specification, CSS3 introduced a modular approach, allowing for the development and adoption of individual modules independently. This approach enabled faster innovation and implementation of new features. CSS3 modules covered a wide range of enhancements, including rounded corners, gradients, transitions, animations, and responsive design capabilities. However, due to the large number of modules, browser support for CSS3 features varied, requiring developers to use vendor-specific prefixes and fallbacks.

In recent years, CSS has continued to evolve, with new features being added and browser support improving. This evolution has focused on enhancing layout capabilities with specifications like Flexbox and Grid, making it easier to create responsive and dynamic web layouts. Additionally, CSS preprocessors and postprocessors, such as Sass and PostCSS, have gained popularity, offering extended functionality and improved workflow for developers.

Overall, CSS has come a long way since its inception, revolutionizing web design and empowering developers to create visually appealing and engaging websites with greater control over presentation and layout.

Basic Syntax

The basic syntax of CSS (Cascading Style Sheets) consists of selectors, properties, and values. Selectors target specific HTML elements, while properties define the styling attributes, and values specify the desired settings for those attributes.

  • Selectors: Selectors determine which HTML elements on a webpage will be styled. They can be based on element types (e.g.<p> for paragraphs), class names (e.g., ".my-class"), IDs (e.g. "#my-id"), or other attributes (e.g. "[attribute="value"]"). Selectors can also be combined or nested to target specific elements or groups of elements. We will look at some of the different types of selectors later on.
  • Properties: Properties define the specific styling attributes that will be applied to the selected elements. Examples of properties include: color for text color font-size for the size of the font margin for spacing around an element and background-color for the background color of an element.
  • Values: Values are assigned to properties and determine how the selected elements will be styled. For instance: color: blue; assigns the color blue to the text. font-size: 16px; sets the font size to 16 pixels. Values can be specified in various units, such as pixels (px), percentages (%), or named colors (red, green, etc.).

    CSS rules are written within curly braces ({}), and each rule consists of a selector followed by one or more property-value pairs. Multiple rules can be grouped together within a CSS file or embedded within an HTML document using the "style" tags.

    • In CSS, you can write a comment by using the /* */ syntax.

Selectors

CSS selectors are like magic words that tell the computer which parts of a webpage to change the colors, sizes, and other things. They help the computer find specific elements, like headings or buttons, to give them special looks. It's like telling the computer, "Make all the buttons blue," or "Make the big titles bold and green." By using these magic words, we can make websites look just the way we want them to.

  • A Universal Selector is like a wild card that matches and selects all elements on a webpage. It uses the asterisk (*) symbol as its selector. When we apply styles to the Universal Selector, those styles will affect every single element on the page. It's like telling the computer, "Hey, change everything in the same way!" This can be useful when we want to apply a certain style globally or reset default styles for all elements. However, we should use it carefully, as it can have a broad impact on the entire webpage. 
    
          * {
          margin: 0;
          padding: 0;
          box-sizing: border-box;
            }
    In this example, the CSS code selects all elements using the Universal selector (*) and applies the following styles:
    • margin: 0; sets the margin of all elements to zero.
    • padding: 0; sets the padding of all elements to zero.
    • box-sizing: border-box;sets the box-sizing property of all elements to "border-box," which ensures that the total width and height of an element include its padding and border.
  • A Type Selector selects elements based on their HTML tag names. It's like telling the computer, "I want to style all elements of a specific type." For example, if we use the selector p, it will target and style all paragraph elements on the webpage. Type Selectors are handy when we want to apply a style to a particular type of element consistently throughout the page, such as changing the font or color of all headings. By specifying the tag name, we can easily target and modify elements of that type to create a cohesive and uniform design. 
    
          p {
            color: blue;
            font-size: 16px;
            font-family: Arial, sans-serif;
            }
    In this example, the CSS code selects all

    elements and applies the following styles:

    • color: blue; sets the text color of all <p> elements to blue.
    • font-size: 16px;sets the font size of all <p> elements to 16 pixels.
    • font-family: Arial, sans-serif; sets the font family of all <p> elements to Arial or any sans-serif font.
  • A Class Selector is like a special label that we can give to certain elements on a webpage. It's like saying, "Hey, I want to style only elements with this special label!" We create a class by adding a dot (.) before its name, like . We can then apply styles specifically to elements with that class by using the class selector. Class Selectors are useful when we want to style specific elements separately from others, like giving a unique color or background to certain buttons or creating a specific style for a particular section.

    HTML:

                <p class="my-class">This paragraph has the class "my-class".</p>

    CSS:

                  
                .my-class {
                  color: red;
                  font-weight: bold;
                  }

    In this example, the CSS code selects all elements with the class name my-class and applies the following styles:

    • color:red;sets the text color of elements with the class my-class to red.
    • font-weight: bold;sets the font weight of elements with the class my-class to bold.
    We can use the same class on multiple elements, allowing us to apply consistent styles to them all at once.
  • An ID Selector is like a special name given to a specific element on a webpage. It's like saying, "Hey, I want to style this one particular element!" We create an ID by adding a hash (#) before its name.

    HTML: 

              <p id="my-id">This paragraph has the ID "my-id".</p>

    CSS: 

                
              #my-id {
                color: blue;
                font-size: 18px;
              }

    In this example, the CSS code selects the element with the ID my-id and applies the following styles:

    • color: blue;sets the text color of the element with the ID my-id to blue.
    • font-size: 18px; sets the font size of the element with the ID my-id to 18 pixels.
    Like #my-id , IDs are unique and should only be used once on a page. We can apply styles specifically to that element using the ID selector. ID Selectors are helpful when we want to style a specific element with a unique design or behavior, such as giving a particular section a distinct background color or applying special formatting to a specific heading.
Common Properties

While there are over 100 different CSS properties, it is very likely that you will work with these ones the most:

  1. color: It's the color of the text you see on a website, like choosing a crayon color for your words.
  2. background-color:It's the color behind the text or the whole page, like picking a color to paint the background of your drawing.

    3. Both color and background-color can accept one of several kinds of values:

    • A keyword, such as an actual color name like red or yellow
    • HEX
    • RGB
    • HSL
      • 
              p{
                /*keyword example*/
                color: red;
                /* hex example */
                color: #1100ff;
              /* rgb example: */
                color:rgb(0, 100, 127);
               /* hsl example: */
                color: hsl(15, 82%, 56%);
              }
  3. font-size: determines how big or small the text appears, similar to choosing the size of your letters when you write
  4. margin: controls the space around an element, like adding extra room between objects on a shelf.
  5. padding: controls the space inside an element, like the empty space you leave around your drawing on a piece of paper.
  6. width: sets how wide an element should be, like stretching or shrinking a rectangle to make it wider or narrower.
  7. height: sets how tall an element should be, like stretching or shrinking a rectangle to make it taller or shorter.
  8. border: creates a line around an element, like drawing a frame around a picture.
  9. display: determines how an element should be shown, like choosing whether to hang a painting on the wall or place it on a table.
  10. position: decides where an element should be placed on the web page, like arranging objects in different spots on a table.
Linking our HTML and CSS

To add CSS to your HTML, you have a few options:

  1. Inline CSS: You can add CSS directly to HTML elements using the "style" attribute. For example: 
                <p style="color: red; font-size: 18px;">This is a red paragraph with a font size of 18 pixels.</p>
  2. Internal CSS: You can include CSS code within the <style> tags in the <head> section of your HTML file. For example: 
                <head>
              <style>
                p {
                  color: red;
                  font-size: 18px;
                }
              </style>
            </head>
            <body>
              <p>This is a red paragraph with a font size of 18 pixels.</p>
            </body>
  3. External CSS: Create a separate CSS file with a .css extension, like "styles.css". Then, link it to your HTML file using the <link> tag in the <head> section. For example:
    4.               <head>
                <link rel="stylesheet" href="styles.css">
              </head>
              <body>
                <p>This is a red paragraph with a font size of 18 pixels.</p>
              </body>
